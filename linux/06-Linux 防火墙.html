Linux 防火墙技术演进

📜 三代防火墙技术

🔥
iptables（RHEL 6 及之前）

基于 netfilter，命令复杂，规则即时生效无持久化。修改规则需全部重载，易中断现有连接。

🛡️
firewalld（RHEL 7+）

动态防火墙管理器，支持运行时/永久双模式，区域概念简化配置，后端仍基于 netfilter/nftables。

⚡
nftables（RHEL 9 默认后端）

替代 iptables，语法更简洁，性能更好，统一 IPv4/IPv6 管理。firewalld 在 RHEL 9 底层使用 nftables。

✨ 为什么选择 firewalld？

1
动态更新

添加规则不中断现有连接，无需全部规则重载

2
区域（Zone）概念

预定义安全策略模板，按网络位置分配信任级别

3
运行时 vs 永久

支持测试配置，确认无误后再持久化，避免服务中断

4
D-Bus API 集成

应用可通过 API 请求临时开放端口（如 UPnP、虚拟机）

🔧 架构层次

用户层 firewall-cmd / firewall-config
↓ D-Bus API
守护层 firewalld（动态管理规则）
↓ 内核接口
后端层 nftables（RHEL 9）
↓ netfilter
内核层 数据包过滤框架

firewalld 九大安全区域

区域 信任度 默认策略 典型应用场景
drop 最低 丢弃所有传入，无回应 防扫描、蜜罐环境
block 极低 拒绝所有传入，返回 ICMP 错误 临时阻断可疑 IP
public 低 仅允许 SSH、DHCP、ping 公共场所 WiFi、酒店网络
external 中低 允许 SSH，默认开启 MASQUERADE NAT 网关外网接口
dmz 中 允许 SSH，限制其他服务 非军事区服务器
work 中高 允许 SSH、Samba、IPP 打印 企业办公网络
home 高 允许 SSH、Samba、DHCP、mDNS 家庭网络
internal 很高 允许大多数内部服务 内网数据中心
trusted 最高 允许所有流量 完全信任环境（慎用）

🎯 区域匹配优先级

  1. 源地址绑定（source）
  2. 接口绑定（interface）
  3. 默认区域（default）

🔧 查看默认区域

firewall-cmd --get-default-zone
firewall-cmd --get-active-zones

⚠️ 安全建议

生产环境严禁使用 trusted 作为默认区域。外网接口必须绑定到 external 或 public。

firewalld 常用命令速查

📊 状态与区域管理

# 查看防火墙状态
firewall-cmd --state

# 查看默认区域
firewall-cmd --get-default-zone

# 查看生效区域及绑定接口
firewall-cmd --get-active-zones

# 查看指定区域所有配置
firewall-cmd --list-all --zone=public

# 设置默认区域
firewall-cmd --set-default-zone=public

# 将接口移动到指定区域
firewall-cmd --permanent --zone=external --change-interface=ens224

🔄 运行时 vs 永久模式

运行时（Runtime）

立即生效，仅存在于内存，重启后丢失。用于临时测试。

永久（Permanent）

写入 /etc/firewalld/ XML 文件，需 --reload 生效，重启后保留。

# 标准操作流程：先运行时测试，再永久保存
firewall-cmd --add-service=http # 立即生效测试
firewall-cmd --permanent --add-service=http # 持久化
firewall-cmd --reload # 重载使永久配置生效

🚪 服务与端口管理

# 查看系统预定义服务列表
firewall-cmd --get-services

# 允许/拒绝服务（永久生效示例）
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --remove-service=https

# 开放/关闭端口
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --permanent --add-port=6000-6010/tcp
firewall-cmd --permanent --remove-port=8080/tcp

# 查看已开放的服务和端口
firewall-cmd --list-services
firewall-cmd --list-ports

📜 富规则（Rich Rules）

# 允许特定 IP 访问特定端口
firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.1.100" service name="ssh" accept'

# 拒绝 ping，但允许特定子网
firewall-cmd --permanent --add-rich-rule='rule protocol value="icmp" reject'
firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.10.0/24" protocol value="icmp" accept'

# 限制连接速率（防暴力破解）
firewall-cmd --permanent --add-rich-rule='rule service name="ssh" limit value="2/m" accept'

📝 常用服务与端口对照

ssh → 22/tcp
http → 80/tcp
https → 443/tcp
dhcp → 67-68/udp
dns → 53/tcp+udp
smtp → 25/tcp
mysql → 3306/tcp

基础网络配置与 IP 转发

1

配置主机名

hostnamectl 设置静态主机名

2

配置三块网卡

nmcli 创建连接，分配 IP 与区域

3

开启 IP 转发

sysctl 设置 net.ipv4.ip_forward = 1

💡 关键概念

IP 转发是 NAT 的前提。Linux 内核默认关闭转发，必须显式开启，否则数据包无法在网卡间路由。

# 步骤 1：设置主机名
hostnamectl set-hostname firewall.company.com

# 步骤 2：配置 DMZ 接口 ens160
nmcli connection add con-name dmz type ethernet ifname ens160 \
  ipv4.method manual \
  ipv4.addresses 192.168.1.1/24 \
  connection.autoconnect yes

# 配置内网接口 ens192
nmcli connection add con-name internal type ethernet ifname ens192 \
  ipv4.method manual \
  ipv4.addresses 192.168.10.1/24 \
  connection.autoconnect yes

# 配置外网接口 ens224
nmcli connection add con-name external type ethernet ifname ens224 \
  ipv4.method manual \
  ipv4.addresses 203.0.113.10/28 \
  ipv4.gateway 203.0.113.1 \
  ipv4.dns 8.8.8.8
# 步骤 3：开启 IP 转发（临时 + 永久）
echo 1 > /proc/sys/net/ipv4/ip_forward
echo "net.ipv4.ip_forward = 1" >> /etc/sysctl.conf
sysctl -p

# 验证
cat /proc/sys/net/ipv4/ip_forward # 应输出 1

🎯 区域信任度模型

drop / block
丢弃 / 拒绝

公共场所 · 仅 SSH/DHCP
external / dmz
外网 / 非军事区
work / home / internal

🔑 核心命令

firewall-cmd --permanent \
  --zone=<zone> \
  --add-interface=<iface>

# 将接口分配到对应区域（持久化）

# 外网接口 → external 区域
firewall-cmd --permanent --zone=external --add-interface=ens224

# DMZ 接口 → dmz 区域
firewall-cmd --permanent --zone=dmz --add-interface=ens160

# 内网接口 → internal 区域
firewall-cmd --permanent --zone=internal --add-interface=ens192

# 重载生效
firewall-cmd --reload

# 验证区域分配
firewall-cmd --get-active-zones

SNAT 配置：内网 / DMZ 共享上网

🔄 SNAT 工作原理

1 内网主机 192.168.10.50 发起访问 8.8.8.8
2 数据包到达网关，将源 IP 替换为公网 IP 203.0.113.10
3 服务器响应到 203.0.113.10，网关查 NAT 表还原目的 IP
4 响应包转发回内网主机 192.168.10.50

💡 MASQUERADE vs SNAT

MASQUERADE：动态使用出口接口 IP，适合 DHCP/拨号

SNAT：固定指定 IP，性能略高，适合静态公网 IP

# 对外网区域开启地址伪装（MASQUERADE）
firewall-cmd --permanent --zone=external --add-masquerade

# 配置 DMZ 区域服务策略
# 允许 DMZ 访问外网 HTTP/HTTPS（软件更新）
firewall-cmd --permanent --zone=dmz --add-service=http
firewall-cmd --permanent --zone=dmz --add-service=https

# 允许内网访问 DNS（解析域名）
firewall-cmd --permanent --zone=internal --add-service=dns

# 重载生效
firewall-cmd --reload

# 验证 MASQUERADE 状态
firewall-cmd --zone=external --query-masquerade # 输出 yes

✅ 验证方法

# 在内网 PC 或 DMZ 服务器上测试
ping 8.8.8.8 -c 4
curl http://www.example.com

# 在网关外网接口抓包验证源 IP 转换
tcpdump -i ens224 -n icmp

DNAT 配置：端口转发与外部访问

🎯 三大转发场景

场景一 Web 服务转发

外网 80/443 → DMZ 服务器 192.168.1.100

场景二 SSH 跳板机

外网 2222 → 内网管理机 192.168.10.10:22

场景三 数据库远程

限定 IP 访问 3306 → DMZ 数据库 192.168.1.102

# ========== 场景一：Web 服务转发 ==========
firewall-cmd --permanent --zone=external \
  --add-forward-port=port=80:proto=tcp:toaddr=192.168.1.100:toport=80

firewall-cmd --permanent --zone=external \
  --add-forward-port=port=443:proto=tcp:toaddr=192.168.1.100:toport=443

# ========== 场景二：SSH 跳板机 ==========
firewall-cmd --permanent --zone=external \
  --add-forward-port=port=2222:proto=tcp:toaddr=192.168.10.10:toport=22

# ========== 场景三：限定 IP 的数据库转发 ==========
firewall-cmd --permanent --zone=external \
  --add-rich-rule='rule family="ipv4" source address="203.0.113.50" destination address="203.0.113.10" forward-port port="3306" protocol="tcp" to-address="192.168.1.102" to-port="3306"'

firewall-cmd --reload

🔍 查看转发规则

firewall-cmd --zone=external --list-forward-ports

🔍 查看富规则

firewall-cmd --zone=external --list-rich-rules

测试验证与故障排查

✅ 四项核心测试

1

内网 PC 访问互联网

验证 SNAT / MASQUERADE 生效

ping 8.8.8.8 && curl http://example.com
2

外网访问 DMZ Web

验证 DNAT 端口转发生效

curl http://203.0.113.10
3

区域隔离验证

外网不可直达内网，DMZ 不可访内网

ping 192.168.10.1 # 应失败
4

SSH 跳板测试

验证非标准端口 DNAT

ssh -p 2222 admin@203.0.113.10

🚨 常见故障排查

内网无法上网

  • 检查 ip_forward 是否为 1
  • 确认外网区域已 --add-masquerade
  • 检查内网 PC 网关是否指向 192.168.10.1
  • 确认接口在正确区域 --get-active-zones

DNAT 转发不生效

  • 检查目标服务器是否监听端口 ss -tlnp
  • 确认目标服务器网关指向防火墙（回程路由）
  • 检查富规则语法是否正确
  • 使用 tcpdump 抓包分析数据包流向

配置未持久化

  • 是否加了 --permanent 参数？
  • 修改后是否执行了 --reload？
  • 检查 /etc/firewalld/zones/ 下 XML 文件
