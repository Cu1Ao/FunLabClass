主机名管理：三种形式与三种方法

📝 三种主机名形式

静态主机名（Static）

存储于 /etc/hostname，系统启动时初始化，内核主机名

瞬态主机名（Transient）

运行时临时分配，由 DHCP/DNS 分配，如 localhost

灵活主机名（Pretty）

UTF-8 格式自由主机名，展示给终端用户，可含特殊字符

💡 关键服务

修改静态主机名后需重启 systemd-hostnamed 服务，使 hostnamectl 识别变更。

# 方法一：nmtui 图形文本界面
nmtui # 选择 "Set system hostname"
systemctl restart systemd-hostnamed

# 方法二：hostnamectl 命令（推荐）
hostnamectl status # 查看当前主机名
hostnamectl set-hostname my.smile.com

# 方法三：nmcli 命令行
nmcli general hostname # 查看
nmcli general hostname RHEL7-1 # 设置
systemctl restart systemd-hostnamed

📋 nmcli 与配置文件对应关系

传统方式 · RHEL7+不推荐
🖥️

nmtui 文本图形

命令行下的伪图形界面，适合无桌面环境远程配置

无桌面首选
⌨️

nmcli 命令行

功能最强大，支持脚本自动化，RHEL 推荐标准方式

企业标准 · 最强大

📄 网卡配置文件关键参数（了解即可，RHEL7+推荐 nmcli）

# /etc/sysconfig/network-scripts/ifcfg-ens33
TYPE=Ethernet
BOOTPROTO=static # none/dhcp
NAME=ens33
DEVICE=ens33
ONBOOT=yes
IPADDR=192.168.10.1
NETMASK=255.255.255.0
GATEWAY=192.168.10.1
DNS1=192.168.10.1

⚠️ 注意：RHEL 7 以后 network 服务已被 NetworkManager 替代，建议使用 nmcli 而非直接编辑配置文件。

nmcli 命令深度解析

🔍 常用查询命令

nmcli connection show # 显示所有连接
nmcli connection show --active # 活动连接
nmcli device status # 设备状态
nmcli device show ens33 # 网卡属性
nmcli connection reload # 重载配置
nmcli connection up/down test2 # 启用/禁用

✏️ 修改连接配置

# 设置为自动启动
nmcli connection modify test2 connection.autoconnect yes

# 修改/添加/删除 DNS
nmcli connection modify test2 ipv4.dns 192.168.10.1
nmcli connection modify test2 +ipv4.dns 114.114.114.114
nmcli connection modify test2 -ipv4.dns 114.114.114.114

# 修改 IP 和网关
nmcli connection modify test2 ipv4.addresses 192.168.10.200/24 gw4 192.168.10.254
nmcli connection modify test2 +ipv4.addresses 192.168.10.250/24 # 辅助 IP

➕ 创建新连接

# 1. DHCP 自动获取
nmcli connection add con-name default type Ethernet ifname ens33

# 2. 静态 IP 配置
nmcli connection add con-name test2 type Ethernet ifname ens33 \
  ipv4.method manual \
  ipv4.addresses 192.168.10.100/24 \
  gw4 192.168.10.1 \
  autoconnect no

# 3. 启用连接并验证
nmcli connection up test2
nmcli device show ens33

# 4. 删除连接
nmcli connection delete test2

💡 核心概念：连接 vs 设备

设备（Device）：物理/虚拟网卡（ens33），由内核管理。
连接（Connection）：对网卡的配置集合。一个网卡可有多个连接，但同时仅一个生效。

网络会话：场景快速切换

🎯 什么是网络会话？

RHEL/CentOS 系统支持网络会话（Network Session）功能，允许用户在多个配置文件中快速切换，非常类似于 firewalld 的区域技术。

🏢
公司网络（company）
静态 IP：192.168.10.1/24，固定网关 DNS
🏠
家庭网络（home）
DHCP 自动获取，动态 IP

⚡ 优势

  • 不同场景一键切换网络配置
  • 配置永久生效，重启后仍然保留
  • 无需手动修改 IP，避免出错
# 1. 创建公司网络会话（静态 IP）
nmcli connection add con-name company ifname ens33 \
  autoconnect no type ethernet \
  ip4 192.168.10.1/24 gw4 192.168.10.1

# 2. 创建家庭网络会话（DHCP）
nmcli connection add con-name home type ethernet ifname ens33

# 3. 查看所有会话
nmcli connection show

# 4. 切换会话（下班回到家）
nmcli connection down company
nmcli connection up home
ifconfig # 验证 IP 已变为 DHCP 获取

# 5. 切回公司网络
nmcli connection down home
nmcli connection up company

📂 配置文件位置

/etc/sysconfig/network-scripts/ifcfg-company
/etc/sysconfig/network-scripts/ifcfg-home

双网卡绑定：负载均衡与高可用

🔗 网卡绑定（Bonding）技术

将多块物理网卡绑定为一块逻辑网卡，实现网络传输速度提升和高可用冗余。即使一块网卡故障，另一块自动顶替，保证业务不中断。

mode0 - balance-rr
轮询策略，依次传输，负载均衡
mode1 - active-backup
主备策略，一块故障另一块接管
mode6 - balance-alb
适配器负载均衡，无需交换机支持 ⭐推荐
# 步骤 1：创建 bond0 接口（mode6）
nmcli connection add type bond con-name bond0 ifname bond0 mode balance-alb

# 步骤 2：为 bond0 配置 IP
nmcli connection modify bond0 ipv4.addresses 192.168.10.1/24
nmcli connection modify bond0 ipv4.method manual
nmcli connection modify bond0 ipv4.gateway 192.168.10.1

# 步骤 3：将 ens33 和 ens37 加入 bond0
nmcli connection add type ethernet con-name slave1 ifname ens33 master bond0
nmcli connection add type ethernet con-name slave2 ifname ens37 master bond0

# 步骤 4：启用绑定
nmcli connection up bond0
nmcli connection up slave1
nmcli connection up slave2

# 验证
cat /proc/net/bonding/bond0 # 查看绑定状态

💡 生产环境意义

7×24 小时服务必须保证网络高可用。mode6 无需交换机做链路聚合配置，兼容性最强，是服务器多网卡场景的首选方案。

SSH 服务基础与配置文件

🔐 SSH（Secure Shell）

以加密方式提供远程登录的协议，是目前远程管理 Linux 系统的首选方式。sshd 是基于 SSH 协议开发的远程管理服务程序。

🔑 基于口令
账户 + 密码验证，简单但存在暴力破解风险
🛡️ 基于密钥
公钥/私钥对验证，安全性高，推荐生产环境使用

📂 关键文件

/etc/ssh/sshd_config # 服务端配置
/etc/ssh/ssh_host_* # 主机密钥
~/.ssh/id_rsa # 客户端私钥
~/.ssh/id_rsa.pub # 客户端公钥
~/.ssh/authorized_keys # 服务端授权公钥
~/.ssh/known_hosts # 已知主机指纹

⚙️ sshd_config 核心参数

参数 作用
Port 22 默认 SSH 服务端口
ListenAddress 0.0.0.0 监听所有 IP 地址
Protocol 2 SSH 协议版本号（仅 v2）
PermitRootLogin yes 是否允许 root 直接登录
PasswordAuthentication yes 是否允许密码验证
PermitEmptyPasswords no 禁止空密码登录
MaxAuthTries 6 最大密码尝试次数
MaxSessions 10 最大并发终端数
# 重启与开机自启 sshd
systemctl restart sshd
systemctl enable sshd

# 基本远程登录
ssh 192.168.10.1 # 默认 root 登录
ssh student@192.168.10.1 # 指定用户
ssh -p 2222 admin@192.168.10.1 # 非标准端口

SSH 密钥验证配置流程

🔄 配置流程

1
服务端创建用户
useradd + passwd 设置密码
2
客户端生成密钥对
ssh-keygen 生成 id_rsa + id_rsa.pub
3
上传公钥到服务端
ssh-copy-id 将公钥写入 authorized_keys
4
服务端禁用密码登录
PasswordAuthentication no，仅允许密钥
5
客户端免密登录
ssh user@host 无需输入密码

✅ 安全优势

  • 避免密码在网络传输中被嗅探
  • 杜绝暴力破解密码攻击
  • 可配合 SSH 代理实现单点登录
# 步骤 1：服务端创建用户（RHEL7-1）
useradd student
passwd student

# 步骤 2：客户端生成密钥对（RHEL7-2）
ssh-keygen # 默认路径 ~/.ssh/id_rsa，可设密钥密码
cat ~/.ssh/id_rsa.pub # 查看公钥内容

# 步骤 3：上传公钥到服务端
ssh-copy-id student@192.168.10.1
# 输入 student 密码后，公钥自动写入 /home/student/.ssh/authorized_keys

# 步骤 4：服务端禁用密码登录（仅密钥）
vim /etc/ssh/sshd_config
# 修改：PasswordAuthentication no
# 可选：PermitRootLogin no（禁止 root 远程登录）
systemctl restart sshd

# 步骤 5：客户端免密登录测试
ssh student@192.168.10.1 # 无需密码直接进入

🔍 验证公钥已上传

cat /home/student/.ssh/authorized_keys

⚠️ 恢复密码登录

# PasswordAuthentication yes
systemctl restart sshd

远程传输命令与 SSH 实用技巧

📤 scp 安全远程传输

基于 SSH 协议的网络文件传输命令，加密传输数据，格式：

scp [参数] 本地文件 远程账户@远程IP:远程目录
▸ 上传文件：scp /local/file.txt root@192.168.10.1:/root/
▸ 下载文件：scp root@192.168.10.1:/root/file.txt /local/
▸ 上传目录：scp -r /local/dir/ root@192.168.10.1:/root/
▸ 指定端口：scp -P 2222 file.txt root@host:/path/

🆚 scp vs rsync

scp 简单直接，适合小文件；rsync 支持增量传输、断点续传，适合大文件或目录同步。

🛠️ SSH 实用技巧

# 1. 禁止 root 远程登录（安全加固）
vim /etc/ssh/sshd_config
PermitRootLogin no
systemctl restart sshd

# 2. 修改默认端口（防扫描）
Port 2222 # 修改后需防火墙放行新端口

# 3. 仅监听内网地址
ListenAddress 192.168.10.1

# 4. 使用 SSH 代理转发（跳板机）
ssh -A jump@bastion.host # 代理密钥转发

# 5. 本地端口转发（隧道）
ssh -L 3306:localhost:3306 user@remote # 将远程 MySQL 映射到本地

🔥 安全最佳实践

✓ 禁用 root 密码登录
✓ 使用密钥验证
✓ 修改默认 22 端口
✓ 启用防火墙限制来源 IP
✓ 设置 MaxAuthTries 限制
✓ 定期更换密钥对
