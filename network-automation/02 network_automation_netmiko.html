4.1 模块介绍

为什么需要 Netmiko？

Paramiko 很好，但网络设备需要更简单的交互

❌ Paramiko 的痛点

手动处理换行符

每条命令后需手动加 \n

手动处理模式切换

需手动发送 system-view、return、quit

手动添加 time.sleep

无法自动判断设备响应是否完成

回显提取困难

原始字符串处理，无结构化解析

✅ Netmiko 的优势

自动处理换行

send_command() 自动追加换行符

自动模式切换

send_config_set() 自动 system-view + return

智能等待回显

自动等待设备提示符，无需 time.sleep

多厂商抽象

device_type 自动适配不同厂商命令语法

核心定位：Netmiko 基于 Paramiko 开发，专门为网络设备 SSH 交互而设计的上层封装库

4.2 核心方法

ConnectHandler

Netmiko 的核心类 · 两种传参方式

方式A

直接传入参数

from netmiko import ConnectHandler

net_connect = ConnectHandler(
  device_type="huawei",
  ip="192.168.56.100",
  username="python",
  password="Huawei12#$",
  port=22,         # 可选
  timeout=10          # 可选
)
方式B

字典解包传参 ⭐推荐

from netmiko import ConnectHandler

SW1 = {
  "device_type": "huawei",
  "ip": "192.168.56.100",
  "username": "python",
  "password": "Huawei12#$"
}

# 注意：使用 ** 解包字典
net_connect = ConnectHandler(**SW1)

📋 ConnectHandler 参数说明

参数必填说明
device_type是设备类型（如 huawei、cisco_ios）
ip / host是设备IP地址或主机名
username是SSH用户名
password是SSH密码
port否SSH端口（默认22）
secret否特权模式密码（思科）
use_keys / key_file否公钥认证相关
conn_timeout否连接超时时间

4.2 核心方法

send_command()

发送查询/排错命令 · 自动等待完整回显

🎯 基本用法

from netmiko import ConnectHandler

dev = {
  "device_type": "huawei",
  "ip": "192.168.56.100",
  "username": "python",
  "password": "Huawei12#$"
}

con = ConnectHandler(**dev)
output = con.send_command("display arp")
print(output)
con.disconnect()

🔍 expect_string 参数

指定预期字符串，读到则立即返回（默认等提示符）

# 读到 "Total:" 就返回，不必等提示符
output = con.send_command(
  "display arp",
  expect_string="Total:"
)

# 如果没读到 expect_string，抛出 OSError

📋 方法特性

✅ 自动追加换行

无需手动加 \n

✅ 自动等待回显

等待设备提示符出现

✅ 自动处理分页

自动发送空格翻页

⚠️ 超时异常

超时返回 OSError

🎯 适用命令

display / show / save 等

4.2 核心方法

配置设备方法

send_config_set() · send_config_from_file()

方法1

send_config_set()

发送一条或多条配置命令，自动进入/退出配置模式

# 单条配置
con.send_config_set("vlan 10")

# 多条配置（推荐用列表）
config_cmds = [
  "vlan 10",
  "int g0/0/10",
  "port link-type access",
  "port default vlan 10"
]
output = con.send_config_set(config_cmds)
print(output)

✅ 自动处理：自动加 system-view 和 return，无需手动输入

方法2

send_config_from_file()

从配置文件读取命令批量下发，适合大量配置

# config.txt 内容示例：
# vlan 10
# int g0/0/10
# port link-type access
# port default vlan 10

output = con.send_config_from_file(
  config_file="config.txt"
)
print(output)

⚠️ 注意：配置文件中不要加入 system-view 和 return

system-view

→

你的配置命令

逐条执行

→

return

自动添加

方法补充

其他常用方法

⏱️ send_command_timing()

基于时间延迟等待回显，而非等待提示符

output = con.send_command_timing(
  "display version",
  last_read=3.0 # 等待3秒
)

适用：命令执行时间不确定的场景

🔌 disconnect()

优雅地关闭SSH连接，释放资源

# 使用完毕后务必断开
con.disconnect()

# 或用上下文管理器
with ConnectHandler(**dev) as con:
  output = con.send_command(...)

建议：使用 with 语句自动管理连接

exit_config_mode() 退出配置模式
save() 保存配置（厂商适配）

实验三

多设备 OSPF 配置

循环批量配置 · SW1 + R1 + R2

from netmiko import ConnectHandler

# 定义3台设备
sw1 = {"device_type": "huawei", "ip": "192.168.56.100", ...}
r1 = {"device_type": "huawei", "ip": "192.168.56.101", ...}
r2 = {"device_type": "huawei", "ip": "192.168.56.102", ...}

devices = [sw1, r1, r2]

# 每台设备的配置
sw1_conf = ["ospf 10", "area 0", "network 192.168.56.0 0.0.0.255"]
r1_conf = ["int loopback0", "ip address 1.1.1.1 255.255.255.255",
           "ospf 10", "area 0", "network 192.168.56.0 0.0.0.255",
           "network 1.1.1.1 0.0.0.0"]
r2_conf = ["int loopback0", "ip address 2.2.2.2 255.255.255.255",
           "ospf 10", "area 0", "network 192.168.56.0 0.0.0.255",
           "network 2.2.2.2 0.0.0.0"]

confs = [sw1_conf, r1_conf, r2_conf]

# 循环批量配置
for i in range(len(devices)):
  con = ConnectHandler(**devices[i])
  out = con.send_config_set(confs[i])
  print(out)
  con.disconnect()

R2 192.168.56.102/24

技术选型

Paramiko vs Netmiko

何时用哪个？根据场景选择合适工具

对比维度 Paramiko Netmiko 建议
定位 通用SSH库 网络设备专用 网络运维选Netmiko
代码量 较多（手动处理交互） 较少（自动处理） 快速开发选Netmiko
多厂商 需自行适配 内置支持 异构网络选Netmiko
配置下发 需手动切模式 自动system-view 配置场景选Netmiko
分页处理 需手动处理More 自动翻页 长输出选Netmiko
灵活性 更高（底层控制） 封装层较厚 特殊需求选Paramiko
学习曲线 较陡 平缓 初学者选Netmiko

✅ 选择 Netmiko 当...

  • • 主要操作网络设备（交换机/路由器/防火墙）
  • • 需要快速开发自动化脚本
  • • 需要批量配置下发
  • • 需要处理不同厂商设备

✅ 选择 Paramiko 当...

  • • 操作Linux服务器/云主机
  • • 需要底层SSH传输控制
  • • 需要自定义SFTP逻辑
  • • 需要端口转发等高级功能

排错指南

常见问题与解决

❌ NetmikoTimeoutException

连接超时，无法建立SSH会话

解决：

检查设备IP可达性、SSH服务是否开启、防火墙是否拦截

❌ AuthenticationException

用户名或密码错误

解决：

核对用户名密码、检查AAA用户service-type是否为ssh

⚠️ OSError / Pattern not detected

send_command 未读到预期提示符

解决：

增加delay_factor、检查设备回显是否含特殊字符、用send_command_timing替代

⚠️ 配置未生效

send_config_set 后配置丢失

解决：

华为设备需执行 save 命令保存配置，或检查权限是否足够（level 3）

💡 device_type 选择

不确定该用什么设备类型

解决：

华为VRP5/6用 huawei，VRP8用 huawei_vrpv8，查官网PLATFORMS列表

💡 CE12800 不支持

连接CE12800报错

解决：

CE12800使用 "system-view immediate" 和 "commit" 模式，Netmiko暂不支持，改用Paramiko
